# Standards Scope and Inheritance

## Standards are not always org-wide
Operating standards often apply at narrower scopes:
- a bar opening standard for bar operation units only
- a seafood receiving standard for seafood categories only
- a peer-group standard for banquet kitchens

## Scope interaction
Policies, benchmarks, and standards are separate layers:
- policies define rule values and thresholds
- benchmarks define peer baselines
- standards define approved operating practices

A standard may influence how memo urgency is routed or how recommendations are assigned, but it is not itself a threshold or benchmark.

## Inheritance model
New locations or operation units may inherit standards from:
1. organization baseline
2. operation-unit-type baseline
3. peer-group standard set
4. local approved override

Inheritance should be explicit and reviewable. Local adoption should not silently rewrite upstream standards.

## Why this matters
Scoped standards let FIFOFlow ask:
- which operating rule applies here
- who owns review for this scope
- should recommendation urgency be elevated because a standard exists and is being missed
